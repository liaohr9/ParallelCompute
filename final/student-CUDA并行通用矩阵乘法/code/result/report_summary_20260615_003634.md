## 可用于实验报告的观察

1. 线程块大小方面，`32x8` 在所有矩阵规模上都取得最优性能，平均性能也最高；`16x16` 次之。过小的 `8x8` 并行粒度不足，过大的 `32x32` 每块线程数达到 1024，调度和资源占用压力更大。
2. 访存方式方面，普通 `global2d` 明显快于 `global2d_btrans`。本实验中 `global2d` 对 B 矩阵按列访问，但同一 warp 内线程访问连续列，访存合并效果较好；`global2d_btrans` 虽然让单线程内连续访问转置后的 B，但 warp 内不同线程访问跨度较大，整体性能反而下降。
3. 共享内存分块 `shared_tiled` 相比 `global2d` 有数据复用，但需要额外同步和共享内存加载；在本机 RTX 5090 和当前实现下，最高性能低于 `global2d`，但仍明显快于较差访存模式。
4. 任务划分方面，`global1d` 将 C 的元素线性分配给线程，实现简单，但平均性能低于二维划分；二维划分更贴合矩阵行列结构，也更容易获得较好的访存合并。
5. 矩阵规模增大后，GPU 利用率更充分，最优 GFLOP/s 从 512 规模的约 5804.93 提升到 2048 规模的约 7681.66。
